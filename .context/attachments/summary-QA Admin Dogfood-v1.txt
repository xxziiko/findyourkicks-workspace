    1.  **Client-side parameter serialization**: The `period` object was not flattened correctly by Axios.
    2.  **Client-side parameter filtering**: `useSearchProducts` only sent `dirtyFields`, omitting default date ranges.
    3.  **Server-side date parsing**: The Edge Function's `new Date()` conversion failed for `YYYY.MM.DD` format.
    4.  **Server-side empty string handling**: The RPC treated empty strings for optional filters as actual filter values (`WHERE status = ''`) instead of `null` (no filter).
    5.  **Client-side Zod schema mismatch**: The client-side Zod schema expected an `id` field, but the RPC returned `product_id`, causing validation failures.
    Each aspect was identified and fixed iteratively, from client-side parameter flattening to server-side date formatting and null/empty string handling, and finally client-side schema adjustment.
- **Transactional Safety (Code Review)**: Initial code review identified a risk of data inconsistency due to non-atomic database operations across multiple API routes (order cancellation, return approval/request). Upon closer inspection, it was determined that most critical operations were *already* using PostgreSQL RPCs (e.g., `cancel_order_after_toss`, `approve_return_request`, `request_return_for_order`), which inherently provide atomicity. Thus, no further changes were needed for these, except for the orphaned image issue.
- **Orphaned Images (Code Review)**: The review highlighted that image uploads preceding database writes could lead to orphaned images if the latter failed. A solution was implemented by adding a `DELETE` endpoint for uploaded images and integrating an `onError` callback in the frontend `ReviewForm` to call this cleanup endpoint if review creation fails.
- **CI/CD Strategy Implementation**: The user's request for PR-scoped and nightly full E2E testing required integrating Playwright tags with GitHub Actions' `paths-filter` to dynamically determine which tests to run. This involved creating new workflows and enhancing existing ones, and debugging server startup conflicts in CI environments.
- **`agent-browser` QA Script Reliability**: Initial `agent-browser` QA runs failed due to assertion mismatches with the actual UI text/structure. This was addressed by carefully inspecting the UI components and updating the shell scripts to use more robust selectors and accurate text for assertions.
