## Priority 1 — 트랜잭션 (Codex 설계 기반)

### 핵심 원칙
- **외부 API(Toss)는 트랜잭션 밖** → 성공 후 단일 RPC 호출로 DB 반영
- **멱등키(cancel_request_id)** 사용 → 중복 요청 안전 처리

### 공통 에러 핸들링 (Codex 설계)

신규 파일: `apps/shop/src/shared/utils/rpcError.ts`
```ts
export function mapRpcError(error: { code?: string; message?: string }) {
  if (error.code === '23505') return { status: 409, error: '중복 요청' };
  if (error.code === 'P0001') {
    if (error.message?.startsWith('FORBIDDEN:')) return { status: 403 };
    if (error.message?.startsWith('INVALID_STATE:')) return { status: 400 };
    if (error.message?.startsWith('NOT_FOUND:')) return { status: 404 };
  }
  return { status: 500, error: '처리 중 오류' };
}
```

### I1. URL 파싱 취약점 — `[reviewId]/route.ts:91`

신규 파일: `apps/shop/src/shared/utils/storageUtils.ts`
```ts
export function extractStoragePath(publicUrl: string, bucket: string): string | null {
  try {
    const url = new URL(publicUrl);
    // pathname: /storage/v1/object/public/{bucket}/{path}
    const marker = `/public/${bucket}/`;
    const idx = url.pathname.indexOf(marker);
    if (idx === -1) return null;
    return url.pathname.slice(idx + marker.length);
  } catch {
    return null;
  }
}
```
